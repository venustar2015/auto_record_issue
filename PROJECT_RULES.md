# 项目宪法与核心规则 (PROJECT_RULES.md)

**重要提示**：这是本项目的最高指导原则。在生成任何代码或逻辑前，请先阅读此文件。如果需求冲突，以此文件为准。

## 1. 核心逻辑约束 (必须严格遵守)

- **时间记录逻辑**
    - **严禁**使用点击保存或输入完成的时间。
    - 必须监听输入框的 `focus` (激活/获得焦点) 事件，在该回调函数触发的**瞬间**获取系统时间（精确到毫秒）并锁定，作为该条记录的时间戳。
- **地理位置处理**
    - 获取经纬度后，必须调用地图 API 进行反向解析。
    - **超时机制**：必须设置 **5秒** 的超时限制。若 5秒 内未返回解析结果，前端/后端必须立即放弃解析，将地址字段标记为“未知”，防止界面卡死。
    - **显示格式**：必须显示为“省市区县街路*号”格式，并附带一个可点击的地图链接。
- **摄像头硬件选择策略 (跨平台)**
    - **核心目标**：无论操作系统环境，优先使用外接 USB 摄像头，避免使用内置摄像头。
    - **筛选规则**：
        1. 调用设备枚举 API 获取摄像头列表。
        2. **第一优先级**：优先选择设备 `label` (名称) 中包含 `"USB"` 或 `"Camera"` 且**不包含** `"integrated"` (不区分大小写) 的设备。
        3. **第二优先级 (降级)**：如果没有找到符合条件的设备，则选择列表中的第一个可用设备。
        4. **错误处理**：如果没有任何设备可用，必须捕获异常，显示“无法获取图片”，图片信息设为空。
    - **逻辑伪代码示意**：`if (camera.label.toLowerCase().includes('integrated')) skip; else use;`

## 2. 数据模型与 Excel 导出规范

- **Excel 文件结构**
    - **文件名规则**：必须严格按照 `[系统版本]_[车型]_[路线]_[测试名称].xlsx` 的格式拼接。
    - **Sheet 1: 详细信息**
        - 必须包含列：问题编号 (自增1)、车型、挂车、载荷、系统版本、路线、天气、测试人员、司机、日期、时间、地点、经纬度、问题描述、**拍照图片**。
        - **图片处理**：图片必须**嵌入**到 Excel 单元格中，不能仅作为外部链接。
    - **Sheet 2: 汇总信息**
        - 必须包含统计摘要（如：总记录数、按类型分类的统计等）。

## 3. 工作区配置默认值

所有新创建的工作区，除非用户明确修改，否则必须应用以下默认值：

- 测试名称: `版本测试`
- 车型: `HG36-244#`
- 测试人员: `测试科`
- 司机: `宋涛`
- 路线: `武汉-恩施`
- 挂车信息: `5#挂`
- 载荷: `满载`
- 系统版本: `V1.1.4`
- 天气: `晴天`

## 4. 系统架构与存储约束

- **存储路径**
    - 工作区元数据：`./workspace/` 目录下。
    - 记录详情数据：`./issue/{workspace_name}/{date_time}/` 目录下。
- **数据可见性**：所有数据必须对“所有访问该网页的用户”可见（即共享状态），这意味着不能使用 `localStorage`（它是浏览器私有的），必须使用后端文件系统或共享数据库。

## 5. 违禁项 (NEVER)

- 不要随意更改全局样式。
- 不要使用未在本文件中声明的第三方库，除非经过用户同意。
- 不要假设用户会手动输入时间，时间必须是自动获取的。
- 不要使用 `alert()` 弹窗，必须使用 UI 组件库的提示。
- 不要在获取摄像头失败时直接崩溃，必须有错误处理。
