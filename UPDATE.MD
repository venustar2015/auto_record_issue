# 网页功能开发需求文档

## 核心记录功能
* **添加备注**：可用户添加备注（问题描述）。
* **自动记录时间**：点击自动记录当前时间（精确到毫秒），记录的时间为激活用户添加备注输入框的那一刻（因为输入文字也占用了时间，激活的那一刻时间比较准确）。
* **自动获取摄像头图片**：
    * 录制的图片应优先使用不含“integrated”的摄像头。例如 Linux 系统输出如下内容：
        ```bash
        ╰─ v4l2-ctl --list-devices                                                                                      ─╯
        1080P USB Camera: 1080P USB Cam (usb-0000:00:14.0-6):
        	/dev/video2
        	/dev/video3
        
        Integrated Camera: Integrated C (usb-0000:00:14.0-8):
        	/dev/video0
        	/dev/video1
        ```
        应该使用 `1080P USB Camera: /dev/video2` 获取图片信息。
    * 如果多个非“integrated”的摄像头，那就按照排序使用功能正常的。
    * 如果只有“integrated”的摄像头，那就使用该内置的。
    * 如果没有摄像头信息，就显示无法获取图片，图片信息为空。
* **自动获取地图经纬度和具体位置**：
    * 显示具体的地点信息（尽可能精确，如省市区县街路*号）。
    * 超过固定时长（例如5秒）无法获取具体地址就显示未知。

## 记录信息列表展示
* 增加记录的信息（包含时间、GPS信息、地理信息和截图信息）的列表。
* **显示模式切换**：可以切换缩略图或者详细信息显示每条记录信息。
    * **详细信息**：包含问题编号、问题时间点、问题描述、经纬度、地图跳转链接（注明具体的地理位置）、图片大图。
    * **缩略图**：包含问题编号、问题时间、问题描述、地图跳转链接、图片缩略图。
* **删除功能**：增加记录删除按钮。

## 工作区功能
* **工作区配置内容如下**：
    * 工作区名称（默认按照“工作区N”形式命名，N为1开始的整数递增，支持名称查重）
    * 测试名称（默认为“版本测试”）
    * 车型（默认为“HG36-244#”）
    * 测试人员（默认为“测试科”）
    * 司机（默认为“宋涛”）
    * 路线（默认为“武汉-恩施”）
    * 挂车信息（默认为“5#挂”）
    * 载荷（默认为“满载”）
    * 系统版本（默认为“V1.1.4”）
    * 天气（默认为“晴天”）
* 支持添加多个工作区。
* 有工作区列表，显示工作区的基本信息，方便用户选择工作区。
* 增加工作区删除按钮。

## 导出功能
* 增加导出功能，将所有的记录信息导出到 excel 文件。
* **文件命名**：excel 名称按照 `"[系统版本]_[车型]_[路线]_[测试名称].xlsx"` 进行命名，如 `“V1.1.4_HG36-244#_武汉-恩施_版本测试.xlsx”`。
* **Sheet 结构**：输出包含【详细信息】和【汇总信息】两个 sheet。
    * **【详细信息】包含**：
        * 问题编号（从1开始自增1编号）
        * 车型
        * 挂车
        * 载荷
        * 系统版本
        * 路线
        * 天气
        * 测试人员
        * 司机
        * 日期（2026/05/27）
        * 时间（10：22：08.326）
        * 地点（省市区县街路*号）
        * 经纬度
        * 地图跳转链接
        * 问题描述（记录信息）
        * 拍照图片（图片内内嵌到单元格）
    * **【汇总信息】包含**：
        * 测试名称
        * 车型（加挂车信息）
        * 测试人员
        * 司机
        * 路线
        * 载荷
        * 系统版本
        * 天气
        * 记录问题数

## 网页记忆功能
* 增加网页记忆功能，记忆力包含如下（对所有进入该网页的用户可见）：
    * **记忆所有工作区信息**（最后一次使用工作区、所有保存工作区），工作区记忆信息保存到 `index.xml` 同级目录下的 `workspace` 文件夹下，某个工作区删除后，对应的记忆信息也会删除。
    * **记忆所有保存的记录信息**，记录记忆信息保存到 `index.xml` 同级目录下的 `issue/workspace_name/date_time` 文件夹下，保存内容包含 excel 中【详细信息】的内容，某个记录删除后，对应的记忆信息也会删除。

## 开发架构要求
* 采用模块化开发的方式，不同功能使用不同的文件，优先将整体框架搭建好，方便后续扩展细分功能。
* 整体框架和各个模块的交互模式明确输入和输出。
* 各个模块和函数写清楚功能注释。
* 为了防止逻辑漂移，请增量编写“单元测试”开发用例。
